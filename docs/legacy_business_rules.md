# 旧系统业务规则提取（Legacy Business Rules）

> 依据：旧项目 `transaction_service.py`、`transaction_types.py`、`view_receivable.py`、`view_budget.py`、`view_stock*.py`、`stock_*.py`、`ai_action_spec.py`。
> 目的：提取不可丢失的业务规则，指导新架构实现。

## 一、交易类型与余额方向

### 交易类型枚举（transaction_types.py）
| 类型 | 旧代码标识 | 说明 | 余额影响 |
|---|---|---|---|
| 收入 | `income` | 普通收入 | 资产账户增加 |
| 支出 | `expense` | 普通支出 | 资产账户减少 |
| 转账 | `transfer` | 账户间转账 | 转出减少，转入增加 |
| 垫付/借出 | `lend` | 应收垫付 | 资产减少，应收增加 |
| 收回欠款 | `recover` | 收回欠款 | 应收减少，资产增加 |
| 余额调整 | `balance_adjustment` | 账户余额修正 | 单账户调整 |
| 股票月结盈利 | `stock_profit` | 投资盈利入账 | 资产增加 |
| 股票月结亏损 | `stock_loss` | 投资亏损入账 | 资产减少 |

### 余额方向规则
- **资产账户**：收入 (+) 支出 (-)
- **负债账户**：负债增加 (+) 负债减少 (-)，余额显示为负数语义
- **应收账户**：垫付 (+) 收回 (-)
- **投资联动账户**：仅由股票模块操作，普通记账不可见

## 二、金额守恒规则

### 交易明细行配对规则
1. **收入交易**：1行收入分类(credit) + 1行资产账户(debit)
2. **支出交易**：1行支出分类(debit) + 1行资产账户(credit)
3. **转账交易**：1行转出账户(credit) + 1行转入账户(debit)
4. **垫付交易**：1行应收账户(debit) + 1行资产账户(credit)
5. **收回交易**：1行资产账户(debit) + 1行应收账户(credit)
6. **余额调整**：1行目标账户(debit或credit)
7. **股票月结**：盈利1行收入分类 + 1行资产账户；亏损1行支出分类 + 1行资产账户

### 守恒约束
- 每笔交易的明细行 signed_amount 之和必须为 0
- 每行必须有且仅有一个账户
- 收入/支出行必须有对应分类
- 转账行不需要分类

## 三、应收账款规则

### 业务规则（view_receivable.py）
- 垫付时创建应收账款记录，关联核心交易
- 收回欠款时可部分收回
- 支持批量核销（多笔应收 → 一笔收回）
- 折损/坏账需明确记录
- 期初应收账款可通过CSV导入
- 应收账户不出现在普通"记一笔"的账户列表中
- 期初数据使用系统分类，不计入普通消费统计

### 状态管理
- 应收记录状态：pending → partially_recovered → fully_recovered / written_off
- is_cleared 标记位兼容旧逻辑

## 四、预算规则

### 预算口径（view_budget.py）
- 按月度管理预算
- 分为收入预算和支出预算两棵树
- 预算绑定到具体分类（Category）
- 实际值从已过账交易中按分类汇总计算
- 预算关闭后保留快照（计划金额、实际值、剩余、超支）
- 关闭的月份不可修改预算数据（需先重新打开）
- 支持从上月复制预算结构
- 支持 AI 生成预算建议

### 统计口径
- 排除系统分类和余额调整
- 排除应收期初修正
- 包含股票月结形成的普通收入/支出

## 五、股票/投资规则

### 资金池规则（stock_*.py）
- INITIAL：初始资金
- TRANSFER_IN：银证转入（核心资产→股票资金池）
- TRANSFER_OUT：银证转出（股票资金池→核心资产）
- ADJUSTMENT：资金调整
- SETTLEMENT：月度结算影响

### 联动规则
- 银证转入：核心资产账户减少 + 股票资金池增加（双向记录，互引ID）
- 银证转出：股票资金池减少 + 核心资产账户增加
- 普通流水不可直接操作投资联动账户
- 删除/修改联动操作必须同步处理两边

### 交易规则
- 买入：资金池现金减少，增加持仓
- 卖出：资金池现金增加，减少持仓（默认不允许卖空）
- 股息：资金池现金增加
- 手续费/税费计入现金计算

### 持仓计算
- 基于交易流水重算数量
- 成本 = 买入总金额（含手续费）/ 数量
- 已实现收益 = 卖出收入 - 对应买入成本 - 手续费
- 未实现收益 = 当前价格 × 持仓 - 持仓成本

### 月度结算规则
- 公式：期末总资产 - 期初总资产 - 当月出入金
- 每个投资账户+月份最多一条有效结算
- 盈利生成"股票投资盈利"收入交易
- 亏损生成"股票投资亏损"支出交易
- 重新生成前作废同月旧结算
- 作废月结同步作废关联普通交易
- 月结必须由用户显式操作

### 撤销规则
- 模块可隐藏（数据保留）
- 模块可归档（数据保留但不参与默认统计）
- 按导入批次回滚（撤销整个ImportJob）
- 永久删除前必须：导出备份 + 用户输入确认文字 + 审计记录

## 六、AI 高风险操作规则

### 审批工作流（ai_action_spec.py）
- AI 只输出结构化 ActionProposal，不直接执行
- 高风险动作（add/update/delete transaction）必须先预览再确认
- 所有确认后操作调用统一交易服务
- AI 操作标记 source=ai
- AI 不可执行永久物理删除
- API Key 不存入普通数据库明文

### 风险等级分类
- 低风险：查询流水、汇总统计
- 中风险：生成预算建议、分析报告
- 高风险：记账、修改流水、删除流水、应收操作

## 七、报表统计口径

### 排除规则
- 系统分类交易不计入普通收支
- 余额调整不计入消费统计
- 应收期初修正不计入真实消费
- 转账不计入收入/支出
- 股票资金池操作不计入日常收支

### 包含规则
- 股票月度结算形成的普通收入/支出计入报表
- 股票联动账户不计入普通总资产（由投资模块单独统计）
- 负债账户以负数语义参与净资产计算

## 八、数据保护规则

- 密码不写入日志或明文存储
- API Key 不入普通数据库
- 真实用户账务内容不入测试快照
- 旧.mym文件只读扫描，绝不直接写入
- 迁移前自动备份旧文件
- 迁移失败保留临时库，不切换正式账本
