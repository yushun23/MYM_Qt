# 旧系统输入说明（LEGACY INPUT REQUIRED）

> 本文件说明如何准备旧系统输入，使 MYM2 的数据迁移功能正常工作。

---

## 需要准备的文件

将以下文件放入 `legacy_input/` 目录：

### 1. 旧账套数据库（必需）

- **文件名**：`my_money.mym`
- **格式**：SQLite 数据库文件
- **来源**：旧 MYM 系统的账套文件
- **重要**：使用**副本**，不要移动或删除原始文件

### 2. 旧系统源码（可选，供参考）

- **目录名**：`MYM-main/`
- **内容**：旧 Python + Flet 源码
- **用途**：供开发者参考旧业务逻辑，非运行时依赖

---

## 安全提醒

1. **始终使用副本**：不要将唯一原件放入 `legacy_input/`，以免误操作损坏。
2. **Git 不跟踪**：`legacy_input/` 已在 `.gitignore` 中，文件不会提交到仓库。
3. **只读访问**：迁移工具以只读模式打开旧 `.mym`，不会修改原始文件。
4. **敏感数据**：旧库中的密钥、密码等不会被导入新系统。

## 缺失时的行为

- 如果 `my_money.mym` 不存在：迁移功能将报告缺失，不会尝试猜测表结构。
- 如果 `MYM-main/` 不存在：不影响迁移和新系统运行，仅缺少代码参考。
