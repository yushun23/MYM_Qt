# MYM2 工程契约（PROJECT CONTRACT）

> 版本：1.0.0 | 最后更新：2026-07-04
> 本文档定义 MYM2 项目不可协商的硬约束。所有开发、审查、合并必须以本文档为准。

---

## 1. 技术栈硬约束

| 领域 | 约束 | 说明 |
|------|------|------|
| GUI 框架 | **PySide6 + Qt Widgets** | 主窗口 `QMainWindow`，导航 `QStackedWidget` + 左侧导航栏。**禁止 Flet、Tkinter、Kivy。** |
| ORM | **SQLAlchemy 2.0** | 仅使用 2.0 风格（`select()` / `DeclarativeBase`）。禁止使用 1.x `Query` API。 |
| 数据库 | **SQLite** | 通过 SQLAlchemy 访问；Alembic 是唯一的 schema 版本来源。 |
| Schema 迁移 | **Alembic** | 禁止手动 `ALTER TABLE` 或裸 SQL 建表。 |
| 图表 | **QWebEngineView + 本地 ECharts** | `echarts.min.js` 放 `resources/vendor/`，禁止 CDN，禁止默认跳外部浏览器。 |
| Python 版本 | **3.11 ≤ Python < 3.14** | 推荐 3.12。 |
| 包管理 | **pip + venv** 或等效 | `pyproject.toml` 声明的依赖必须可安装。 |

## 2. 金额规则（不可变）

1. **数据库存储**：所有金额字段类型为 `INTEGER`，单位为**分**（**整数分**方案）（1 元 = 100 分）。
2. **Python 运算**：金额文本解析和业务计算使用 `Decimal`。
3. **禁止**：任何金额业务逻辑使用 `float` 或 SQLite `REAL`。
4. **禁止**：数据库中出现 `REAL`、`FLOAT`、`DOUBLE` 类型的金额列。
5. **UI 显示**：仅在渲染层将分转为元格式化，格式化逻辑集中在统一工具函数中。

---

## 3. 写账安全规则

1. **唯一写入口**：所有会改变账本数据（`accounts`、`transactions`、`categories`、`budget_lines` 等）的动作，只能通过 `LedgerService` 或明确注册的专用服务完成。
2. **禁止直写**：UI 层、导入器、AI 模块、报表模块 **不得** 直接执行 `INSERT`/`UPDATE`/`DELETE` 到 `transactions` 或 `accounts.balance`。
3. **审计**：所有写操作应产生 `AuditEvent` 记录。

---

## 4. 股票/证券模块规则

1. **新项目彻底不实现功能性股票模块**：没有股票导航入口、页面、持仓列表、行情查询、买卖录入、证券导入、月度结算或网络报价。
2. **允许保留**：
   - 通用历史归档表 `legacy_archive_records`（仅存原始数据，不解析）。
   - 不可编辑的"历史投资资产快照"账户（来自迁移）。
3. **禁止**：在 UI 导航中显示"股票""证券""投资组合"等入口。

---

## 5. 旧系统迁移规则

1. **只读打开**：旧 `.mym` 文件必须以 SQLite 只读 URI 模式打开（`file:///path?mode=ro`）。
2. **预检查**：迁移前必须验证旧库完整性、表结构、数据行数。
3. **Dry‑Run**：必须支持不写入的试运行模式，输出迁移报告。
4. **显式确认**：正式迁移前必须获得用户确认。
5. **事务回滚**：迁移在一个 SQLAlchemy 事务中完成；任何失败必须完整回滚。
6. **导入报告**：迁移完成后输出详细报告（导入行数、跳过原因、警告）。
7. **重复防护**：通过 `LegacyIdMap` 防止重复导入。
8. **不确定项**：任何不确定的迁移项只能报告，不能猜测写入。

---

## 6. 安全与秘密管理

1. **禁止持久化秘密**：API key、密码、代理密码、旧 `password_hash` **不得** 写入新数据库、日志文件、测试夹具或 Git 仓库。
2. **AI 凭据**：仅允许使用系统 keyring（`keyring` 库）或当前会话内存。
3. **旧 settings 白名单**：迁移时只导入明确允许的配置项（如主题偏好）；敏感值一律跳过。
4. **日志脱敏**：日志输出不得包含密钥、密码、token。

---

## 7. 数据安全

1. **数据安全优先于功能**：任何可能破坏或污染账本数据的操作必须被拒绝。
2. **备份优先**：迁移前提示用户备份。
3. **只读优先**：查询和报表不得产生副作用。

---

## 8. 测试纪律

1. 每一步开发完成后必须运行该步骤验收测试及全量已有测试。
2. 测试失败必须先修复，不得跳步。
3. 测试数据库必须使用临时文件或内存数据库，不得污染开发/生产数据。
