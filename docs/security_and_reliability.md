# MYM 安全性与可靠性

## 安全措施

### 数据访问
- 所有数据库访问通过 SQLAlchemy 参数化查询，防止 SQL 注入
- 禁止 UI 层直接执行 SQL
- 每个线程/后台任务使用独立短生命周期 Session

### 数据保护
- API Key 和密码绝不写入日志
- 用户真实账本路径不在日志中暴露
- HTML/图表中用户输入在 Python 端转义
- 生产环境不依赖 CDN（本地 ECharts）

### 文件处理
- 文件扩展名白名单校验
- 文件大小限制（附件 10MB，导入 20MB）
- 损坏文件返回明确错误，不崩溃
- 图片预览限制尺寸

### 审计
- 所有资金操作写入审计日志
- AI 操作记录请求摘要和结果
- 敏感字段脱敏

## 可靠性措施

### SQLite 并发
- UI 线程与 worker 线程独立 Session
- 写入序列化，避免锁冲突

### 异常恢复
- 导入/迁移失败自动回滚
- AI 调用失败展示错误，不影响数据
- 关键操作前自动备份

### 数据完整性
- 账务金额重算：opening_balance + 已过账交易
- 外键约束启用
- 数据库 integrity_check 入口

### 性能基准
- 流水分页加载
- 数据库索引覆盖查询路径
- 大账本（10万+交易）性能目标：
  - 流水列表加载 < 1s
  - 月度报表生成 < 3s
  - 余额重算 < 5s
